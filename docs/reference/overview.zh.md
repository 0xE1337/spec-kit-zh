# CLI 参考

Specify CLI（`specify`）管理规范驱动开发的完整生命周期——从项目初始化到工作流自动化。

## 核心命令

创建和管理 Spec Kit 项目的基础命令。用必要的目录结构、模板和脚本初始化新项目；验证系统是否安装了所需工具；查看版本和系统信息。

[核心命令参考 →](core.md)

## 集成

集成把 Spec Kit 连接到你的 AI 编码智能体。每个集成会为特定智能体配置对应的命令文件、上下文规则和目录结构。每个项目同一时间只有一个集成处于活跃状态，你可以随时在集成之间切换。

[集成参考 →](integrations.md)

## 扩展

扩展为 Spec Kit 增加新能力——领域专属命令、外部工具集成、质量关卡等等。扩展通过目录源发现，可以各自独立地安装、更新、启用、禁用或移除。多个扩展可以在同一个项目中共存。

[扩展参考 →](extensions.md)

## 预设

预设定制 Spec Kit 的工作方式——覆盖命令文件、模板文件和脚本文件，而不改动任何工具。它们让你可以强制执行组织标准、让工作流适配你的方法论，或把整个体验本地化。多个预设可以按优先级排序叠加，实现分层定制。

[预设参考 →](presets.md)

## 工作流

工作流把多步骤的规范驱动开发过程自动化为可重复的序列。它们把命令、提示词、shell 步骤和人工检查点串联起来，支持条件逻辑、循环、扇出/扇入，并且能够暂停后从中断的确切位置恢复。

[工作流参考 →](workflows.md)

## 套装

套装把现有的扩展、预设、工作流和步骤组合成一个带版本、可安装的整体单元。套装不添加新行为，而是精选一组基础组件——一个团队或角色所需的一切——并通过每个组件自己的机制一步完成安装，附带版本固定、冲突检查和来源追踪，保证干净的更新与移除。

[套装参考 →](bundles.md)
